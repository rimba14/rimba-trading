PHASE 1 — ARCHITECTURE, SYNCHRONISATION & SCALABILITY
═══════════════════════════════════════════════════════════════════════════════

1.1  Information-Driven Architecture (Dollar/Volume Bars)
     • Time Bars (e.g. 5-min charts) are strictly prohibited.
     • The Dual-Loop matrix is event-driven via gitagent_bars.InformationBarStreamer.
     • Default threshold: $10,000,000 USD per dollar bar.
     • Both Slow Loop (sentinel_slow_loop.py) and Fast Loop (chat_gemma.py) only
       trigger on statistically significant fiat-volume thresholds.
     • System natively sleeps during low-volume noise; hyper-samples during spikes.

1.2  Symbol Auto-Discovery
     • Broker suffixes MUST NEVER be hardcoded.
     • sentinel_config.get_valid_mt5_symbol() implements a 3-phase fallback:
       Phase 1 — direct match | Phase 2 — suffix probe | Phase 3 — full scan.
     • resolve_watchlist() builds WATCHLIST dynamically at boot.

1.3  Universal UTC & Staleness Gate
     • All timestamps written to / read from ArcticDB use utils.get_utc_epoch()
       (UTC epoch seconds).
     • ArcticDB reads AND writes enforce a hard 300 ms timeout via
       concurrent.futures.ThreadPoolExecutor (ARCTIC_TIMEOUT = 0.3).
     • If cached signal age > STALENESS_THRESHOLD (900 s), [STALE_SIGNAL] is
       logged and new entries for that asset are halted.

1.4  Fractional Differentiation (FracDiff)
     • Integer differentiation (pct_change() / .diff()) on raw price series is
       strictly forbidden for ML model inputs.
     • optimize_fracdiff_d() finds the minimum d (step 0.05) achieving ADF
       stationarity (p < 0.05), preserving maximum market memory.
     • Applied independently to all OHLC columns before Kronos/XGBoost/TimesNet.

PHASE 2 — PERCEPTION & COGNITION (LOCAL BRAIN / HEAVY COMPUTE)
═══════════════════════════════════════════════════════════════════════════════

2.1  HMM Oracle
     • gitagent_hmm.get_current_state() returns (state, prob, _).
     • States: BULL | BEAR | RANGE with dynamic std-dev regime boundaries.
     • HMM data cached to ArcticDB: {symbol}_hmm (state, prob, atr, timestamp).

2.2  Meta-Labeling Architecture (Decoupled Sizing)
     • Primary Model (Direction only): Kronos + XGBoost blend (70/30).
       p_blend > 0.55 → LONG (+1) | p_blend < 0.45 → SHORT (-1) | else HOLD (0).
     • Meta-Model (Conviction / Sizing): XGBClassifier (medallion_model.json).
       Outputs probability p that the primary direction is correct.
     • Final conviction: p_final = (base_p × 0.60) + (moe_confidence × 0.40).

2.3  Sovereign MoE Reasoning & Memory Lock
     • qwen_reasoning_engine.QwenReasoningEngine uses the native Ollama /api/generate
       endpoint (not OpenAI-compatible layer).
     • Payload MUST include "keep_alive": -1 to lock the model in RAM.
     • Fail-safe: any Timeout or endpoint error returns HOLD / 0.500 immediately.

2.4  SHAP Feature Importance
     • shap.TreeExplainer(_META_MODEL) runs for every prediction in Slow Loop.
     • Top 3 positive + negative drivers formatted into JSON and written to
       shap_diagnostics/{symbol}_diag.json.
     • Payload fields: symbol, prediction, conviction, reasoning, timestamp,
       weights (all features), top_pos, top_neg, concept_drift (bool).

2.5  Epistemic Gate (0.82 Threshold)
     • norm_p = abs(meta_p - 0.5) + 0.5
     • Signals are ONLY routed to pending_signals/ if norm_p >= 0.82 AND
       primary_dir != 0.
     • This gate is re-enforced on the VPS side (vantage_execute.py).
     • Bypass of this gate is constitutionally prohibited.

2.6  Regime Alignment
     • If HMM == BEAR and primary_dir == BUY  → meta_p forced to 0.50 (blocked).
     • If HMM == BULL and primary_dir == SELL → meta_p forced to 0.50 (blocked).
     • Applied in BOTH slow_loop and vantage_execute for belt-and-suspenders.

PHASE 4 — RISK GATES & FRACTIONAL KELLY SIZING
═══════════════════════════════════════════════════════════════════════════════

4.1  Amnesia Lock
     • Before any execution, check MT5 open positions.
     • If a position in the SAME direction already exists for that symbol
       (magic=142), the new sub-order is blocked.

4.2  Weekend Blackout
     • Forex, Indices, Equities: blacked out Friday 21:55 UTC → Sunday 22:15 UTC
       (≈ Friday 23:55 → Monday 00:15 broker UTC+2 time).
     • Crypto (CRYPTO regime): runs 24/7, never blacked out.

4.3  Fractional Kelly Sizing
     • f_raw  = p − (q / b)   where q = 1−p, b = 2.0 (assumed 2:1 reward:risk).
     • f_kelly = f_raw × KELLY_FRACTION (0.25).
     • Capped at HARD_RISK_CAP = 0.02 (2.0% of equity per trade).

4.4  Small-Account Execution Bypass
     • If 0.0 < calculated_lot < MIN_LOT (0.01): round up to 0.01.
     • Log: "[HARD_RISK_CAP_WARN] … Rounding UP to allow signal execution."

4.5  Absolute Risk Ceilings
     • Portfolio Heat cap:  PORTFOLIO_HEAT_CAP  = 20%  (used_margin / equity).
     • Hard Risk Cap:       HARD_RISK_CAP       = 2.0% (per-trade f_kelly ceiling).
     • Leverage Wall:       LEVERAGE_WALL       = 10×  (total notional / equity).

     • VPS machine protects positions even if Machine A (Laptop) goes offline.
